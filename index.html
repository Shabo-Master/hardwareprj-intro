基于ONetSwitch的家庭网关

基于ONetSwitch开发板的家庭网关，能通过锐捷身份认证，具有远程状态监控功能。将其应用在使用锐捷认证的高校，能实现多设备上网，为使用者带来便利。

简介

该成品具有多个网口，连上互联网之后可作为具有NAT功能的网关使用。同时具有锐捷认证的能力，在使用锐捷认证计费方案的高校具有很高的便利性。另外还具有通过互联网远程状态监控的程序。

成品硬件上使用的是ONetSwitch20开发板。ONetSwitch开发板是基于Xilinx公司的Zynq-7000 ARM-FPGA芯片设计的：

  • 外设十分丰富，具有五个GE口，非常适合作为家庭网关使用
  • ARM-FPGA芯片，具有性能强大的ARM-a9核，作为家庭网关使用绰绰有余
  • ARM-FPGA芯片，硬件上可编程，有一定自主改变硬件的能力
  • 该平台母板是一块ZedBoard，有Xilinx官方资料加成，网上资料丰富

使用嵌入式Linux，OpenWrt的开源项目：

  • OpenWrt作为开源的Linux发行版，获取十分方便
  • 该项目广泛用于家庭路由器，有相当多的功能为家庭路由器设计，给我们的项目带来相当多的便利
  • 提供相当多的软件包供扩展
  • 其官方支持Zynq-7000芯片

成品功能分为三个部分，网关功能、锐捷认证功能、远程状态监控功能。

整个成品非常适合在学校使用，实现自动认证，多设备上网，远程管控。使用时，仅需将网关连上校园网接口，插上电源即可开机自动认证，将上网设备连上网关的GE口即可直接上网，使用管控客户端即可在电脑上监控设备的CPU负载、内存占用、网络等状态，并且对其进行控制。

网关功能实现

使用ONetSwitch官方固件和Bootloader作为启动文件，文件系统使用的是LEDE(OpenWRT)的开源项目提供的文件系统，以此搭建了一个嵌入式Linux系统，编译Linux内核开启Vlan和iptables功能，实现NAT功能。

锐捷认证是锐捷公司私有的认证协议，其认证客户端并不对外开源，因此我们使用的是第三方的Linux认证客户端MentoHUST。使用LEDE提供的工具链单独编译

远程状态监控

远程状态监控能通过网络远程对网关的状态进行获取，并且能对其进行控制。使用Python实现，通过socket套接字传输信息。

技术

嵌入式Linux

嵌入式Linux（Embedded Linux）是一类嵌入式操作系统的概称，这类型的操作系统皆以Linux内核为基础，被设计来使用于嵌入式设备。

ONetSwtich板卡

ONetSwitch板卡是由南京MeshSr公司开发的，以Xilinx公司的Zynq系列芯片作为处理核心的专门用来开发交换机应用的开发板。开发板有着丰富的网络外设资源，可以进行灵活的扩展和二次开发。

OpenWrt

OpenWrt是适合于嵌入式设备的一个Linux发行版。相对原厂固件而言，OpenWrt不是一个单一、静态的固件，而是提供了一个可添加软件包的可写的文件系统。这使用户可以自由的选择应用程序和配置，而不必受设备提供商的限制，并且可以使用一些适合某方面应用的软件包来定制你的设备。

Python

Python，是一种面向对象、直译式的电脑程序语言。它包含了一组功能完备的标准库，能够轻松完成很多常见的任务。

网关实物图

网关实物图

该网关是基于叠锶公司的ONetSwitch20开发板设计的。ONetSwtch20整体上是两部分，母板是Xilinx公司的Zedboard开发板，扩展板是叠锶公司自己设计的，扩展了四个GE口，合计五个GE口。

我们将母板上的GE口作为WAN口连接互联网，其他四个口连接内网。

系统上，初时我们使用官方提供的内核文件，文件系统使用OpenWrt的文件系统。

首次将系统搭建起来之后开启所有网口，发现无法与开发板网络通信，但是单网口没有问题。遂重新编译Linux内核，开启对802.1q的支持，重新配置Vlan后解决。

发现缺少NAT转发功能，内核未支持iptables。同样在编译Linux内核时将ipbales的支持加上。

转发性能测试

转发性能测试

转发测试时使用5线程，双向各测试10次后取平均值。

内网转发是指内网GE口直接转发，不进行NAT。NAT转发是指数据包在内外网之间转发需要在内核中进行包头地址的转换。

内网转发带宽在540Mbps左右，NAT转发330Mbps左右。可以看出NAT转发对内核性能要求会比较高。

成品的整体性能已经足够日常使用了，特别是学校的宿舍区（沁苑除外）有线入网带宽限制是100Mbps，已经足够使用。

锐捷认证
锐捷认证过程

锐捷认证过程

锐捷认证过程与询问握手协议（Challenge-Handshake Authentication Protocol，CHAP）类似。过程中有三个设备参与———用户、交换机、中心服务器。

认证时用户连上交换机并发起请求->交换机询问身份认证->交换机将身份信息给中心服务器->服务器回复->交换机询问用户密码->用户回复密码->交换机将密码给服务器->服务器查询后返回验证结果->交换机返回认证结果->用户认证结束

交换机在认证过程中起到控制用户网络连接状态和认证中介的作用，中心认证服务器则存储用户名、密码等信息，对认证进行集中管理。

远程状态监控
远程控制

远程控制

远程控制使用的socket套接字进行通信。cs结构，开发板作为服务器端，用户控制端作为客户端。输入预设的IP地址和端口连上开发板的服务端。

客户端和服务端连上之后，客户端仅对命令进行采集，通过套接字将其发送到服务端。

服务端使用Python的subprogress模块将字符串输出到命令行进行控制。

可以将该功能进行进一步的扩展，将常用功能做成按键，将更方便使用。

远程状态监控
远程信息获取

远程信息获取

远程控制使用的socket套接字进行通信。cs结构，开发板作为服务器端，用户控制端作为客户端。输入预设的IP地址和端口连上开发板的服务端。

开发板服务端从Linux系统获取系统信息，并将其组织成Dict结构。再将数据转换成字符串发送给客户端。

客户端收到字串流，将其转换回Dict结构，并在界面上进行输出。

远程状态监控
Socket通信

Socket通信

服务器首先启动，通过调用 socket()建立一个套接口，然后 bind()将该套接口和本地地址（IP 地址和端口）绑定在一起，再 listen()使得套接口做好侦听准备，并规定它的请求队列的长度，之后就调用accept()来接收连接，并获得客户 机的地址信息。

客户机在建立套接口之后就可以调用 connect()和服务器建立连接。

连接一旦建立，客户机和服务器之间就可以通过调用 send()和 recv()来发送和接收数据。

最后，待数据传送结束后，双方调用 closesocket()关闭套接口。
